Architecture

There are 5 layers in Driver Core.

  1. Clusters
  2. Servers
  3. Connections
  4. Operations
  5. Bindings

Clusters

Clusters represent a remote set of servers. This set could be 1, which is represented by the SingleServerCluster. It is the cluster’s responsibility to provide Server Selection capabilities. It does this by monitoring each server and, when they change, altering its ClusterDescription.

In addition, clusters are responsible for collaborating with the servers to discover and monitor changes as they happen (see the Server Discovery and Monitoring Specification. These changes could be anything from a node going down to the utilization of a new node added to a replica set.

Servers are provided to a cluster with an IClusterableServerFactory.

Server Selection

Server Selection is implemented by the IServerSelector interface which is provided to the SelectServerAsync method.

Servers

A server represents a one-to-one mapping from a client IServer to a MongoD or MongoS running on the network. Servers are responsible for monitoring themselves and raising the DescriptionChanged event.

Connections are provided to a server with an IConnectionFactory.

Connection Pooling

Connection Pools are managed by the server. The server monitors each in-use connection and, upon certain errors, instructs the connection pool to kill all its connections and start over.

Connection pools are provided to a server with an IConnectionPoolFactory.

Connections

Connections are responsible for communicating the wire protocol over the network. It does this with a Stream provided by an IStreamFactory.
